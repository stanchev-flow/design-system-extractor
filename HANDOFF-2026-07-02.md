- **Hybrid generation**: LLM emits a `composition.v1` structured object (never HTML) → `compose_from_composition.py` adapter → deterministic composers (`compose_section.py`, SSOT renderers in `component_render.py`, page assembly `compose_page.py`) → on-brand gate. ## User's operating rules (hard-won; do not violate)

- **Everything through the live pipeline.** No hand-authored HTML or composition JSON, ever. Deterministic composer previews are OK for showcase rows only.
- **Every visual defect becomes a mechanical rule**: composer discipline (can't emit it) + gate check (fails if emitted) + composition-rules note. "Anti-slop" = no silent fall-throughs; resolution sources explicit and stamped (`data-pattern`, `data-align-source`).
- Slop classes already ruled: bright decoration over text, gold hover on cream (interaction tokens re-scope per surface INCLUDING cards), centered text + offset media, accidental left-alignment, empty containers, duplicate eyebrows, unbound counters, occluded heading lines, heading/text collisions.
- Anchored variation for generative work: seed from measured/known-good, vary one axis, drift-band gate.
- Style files: identical structure across styles, YAML front-matter authoritative, qualitative values only (no px/hex in style layer).
- Versioned lanes for every run; gate-green only in lanes; failures listed in reports with reasons.
- WoodWave brand truths: Melodrama self-hosted (hero 500, section headings 400), gold #edd580 hover on dark only, nav is page-level sibling (not inside #sec-0).

## Worker/process hygiene (learned the hard way)

- Background workers sometimes die silently on connection errors AND sometimes go 60–80 min without disk writes while computing. **Never assume death from silence**: check file mtimes over a 5–10 min window before relaunching; a recovery worker must stop if it sees recurring writes (this protocol saved us once tonight).
- Serialize all writers on `compose_section.py` / `component_render.py` / `compose_page.py` / `onbrand_check.py` / `styles.py`. Parallel-safe: read-only analysis, own-file tasks (preview generator), `runs/<other-brand>/**`-fenced tasks, style-file edits ONLY as atomic whole-file writes.
- Subagent transcripts do not stream reliably; judge progress by disk artifacts.
- `phase0_regate.py` after any composer/gate change; zero PASS→FAIL; rebaseline only documented intended diffs.
