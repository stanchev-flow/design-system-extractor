## 2026-07-17 — STYLE AUTO-RESOLUTION wired into the DEFAULT generate path: presets now shape every generation, not just opt-in lanes (fact-gated, byte-identical, explicit-wins, fail-open)

- `brand_pipeline/generate_composition.py`: the pass-3 stage-2 style block is no
  longer caller-only. `generate_composition()` now resolves the ALREADY-CHOSEN
  `style_id` through the style-library resolver and injects
  `render_style_directive_block` output by default. Wired at the higher-level
  entrypoint (right before `build_prompt`, ~line 1476) via two small helpers —
  `_auto_style_directives()` (load_library → resolve_all → render) and
  `_resolve_style_directives()` (precedence gate) — so `build_prompt` stays a
  pure assembler and the library loads at most once per run. No detection step:
  this auto-RESOLVES the picked id; style DETECTION from a source site remains a
  separate future feature. Precedence + `presetDissents` logging are unchanged
  (owned by the resolver).
- FACT-GATED / BYTE-IDENTITY: auto-resolution is gated on PRESET presence. A
  style with no preset (`dark-mode`, the 1/51 uncovered id) and any non-library
  style id (e.g. the base-only `corporate-saas-clean`) return no block, so the
  assembled prompt stays byte-identical to the pre-wiring assembly. A
  preset-backed id (`swiss`/`bauhaus`/…) gets the resolved block with authored
  preset defaults folded in UNDER any measured brand fact.
- EXPLICIT-WINS / OPT-OUT: an explicit caller `style_directives` is honored
  verbatim (a non-None value wins, including `""` = suppress). Only `None`
  triggers auto-resolution; the opt-out path never touches the library.
- FAIL-OPEN: a missing/unloadable style-library or ANY resolver error degrades
  to no block, no crash (this is a live generation path — prompt shaping must
  never take a run down).
- Prompt-shaping ONLY: no renderer/gate/replica code path was touched. Replica
  SCORES cannot move (confirmed — no rendering path changed; full replica scoring
  NOT re-run). The pass-3 fence (no renderer/gate module imports the resolver)
  stays intact; `generate_composition` was already an allowed consumer.
- Tests: NEW `brand_pipeline/tests/test_style_autoresolve.py` (16) covering the

## 2026-07-09 — studio_server.py: silent-hang root cause found (stack-dump proven), hardened, soak-verified

The Design System Studio server (port 1500) silently hung three times today: process alive, port accepting nothing, no traceback. Root cause reproduced, proven with a faulthandler all-thread dump, fixed, and soak-tested. No commits.

**Root cause (reproduced under `ulimit -n 96`, dump captured via `PYTHONFAULTHANDLER=1` + SIGABRT):** handler connections had NO socket timeout (`StreamRequestHandler.timeout = None`), so any client that connects and stalls — browser preconnects, suspended tabs mid-download (every response is `no-store`, so each `/project/remote` view re-transfers the full ~85MB asset tree), sleep/wake half-open sockets, killed Playwright runs — pins one `ThreadingHTTPServer` handler thread + one fd FOREVER. Once fds ran out, `accept()` raised EMFILE which `socketserver._handle_request_noblock` **swallows silently**; with the default listen backlog of 5 overflowing, macOS RSTs/blackholes new connections → process alive, port dead, zero output. The hinted theories that didn't hold: the only `urlopen` runs in a background job thread (not a handler) and already had `timeout=25`; all `JOBS_LOCK` sections are fast dict ops (the log tail read was already outside the lock).

- **:1235** `StudioHandler.timeout = 30` (env `STUDIO_SOCKET_TIMEOUT_S`) — THE fix. Applied to every connection by `StreamRequestHandler.setup()`; stalled reads/writes now raise `TimeoutError`, http.server closes the connection, thread + fd are reaped. Timeout is per socket op, so slow-but-progressing transfers are unaffected.
- **:1299** `/healthz` — instant, lock-free, no-filesystem liveness probe returning `{ok, pid, uptime_s, threads}`; `threads` is the wedge canary.

- **:2265** `_raise_fd_limit()` — soft `RLIMIT_NOFILE` raised toward 4096 at startup (macOS shells often default to 256; headroom, not the fix).
- **:935-948** `fetch_html` — total wall-clock deadline (env `STUDIO_FETCH_DEADLINE_S`, default 75s) on top of the per-op socket timeout (env `STUDIO_FETCH_TIMEOUT_S`, default 20s), reading via `read1` chunks so a drip-feeding server (1 byte per 2s keeps every socket op "successful") can no longer stall a harvest job forever.
