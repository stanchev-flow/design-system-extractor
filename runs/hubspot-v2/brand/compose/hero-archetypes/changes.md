# hero-archetypes — HubSpot v2 genre-archetype hero gallery (phase 2, 2026-07-13)

Eight heroes, one per page type, each instantiating a DIFFERENT structural skeleton
from the SaaS hero genre library (`brand_pipeline/contracts/archetypes/heroes-saas.yaml`,
law: `brand_pipeline/spec/archetype-library.md` — style-invariant / structure-variable /
physics-hard). Every visible property comes from HubSpot v2's own brand data
(`runs/hubspot-v2/brand/brand.yaml` + layout-library recipes/patterns); only the
skeleton comes from the library. Lane generated by `tools/run_hero_archetype_gallery.py`
with `force_off_grid=True` (the brand's style pin untouched) and honest per-page-type
copy authored in `briefs/*.md`.

## Roster (page type → archetype, why)

| page | archetype | why (affinity) |
| --- | --- | --- |
| homepage | `hero-split-product` | flagship split: promise + product canvas right — the platform lead the source never shows as a split |
| pricing | `hero-pricing-value-forward` | price-transparency stack: value claim first, compact band, plans below the fold |
| product | `hero-product-canvas-panel` | overlay panel on a full product canvas — feature-page shape for Marketing Hub |
| about | `hero-mission-monument` | centered mission statement as monument type over supporting photo |
| blog | `hero-content-featured-lead` | content-hub split: hub promise + featured piece as the lead card |
| demo | `hero-form-split` | lead-gen split with the demo qualifier copy leading toward the form panel |
| developer | `hero-search-first` | docs-entry stack anchored on the search affordance (`hero-terminal-split` excluded — see un-instantiable) |
| event | `hero-announcement-crest` | launch announcement with product crest bleeding into the band seam (off-grid device, sanctioned by the lane's off-grid expansion) |

Novelty: the extracted homepage hero is a `layered-fullbleed-scrim`; **zero of the 8**
reuse that skeleton family (`hero-layered-fullbleed-scrim` is excluded in every brief —
stricter than the "at most one" allowance). `about` and `event` seed treatments from
the extracted `hero-photo-overlay` pattern but instantiate monument/crest skeletons,
not the source's layered collage anatomy.

Un-instantiable with current facts: **`hero-terminal-split`** (developer brief) — needs
a code-surface fact family (monospace stack + code-panel surface/inset facts) that
HubSpot v2's extraction does not carry (no `typography.mono` role, no code-panel
recipe). Excluded via brief `excludeArchetypes`; `hero-search-first` chosen instead.
What it needs: a `code-panel` block contract + a brand-declared mono type role.

## Vocabulary extensions (PHASE2-PLAN §6)

Implemented (schema-documented, fact-gated, tested):

- **`knobs.bandHeight`** (`compact|standard|tall`; `viewport` degrades to `tall`) —
  re-registers the section's vertical padding to the NEAREST rung of the brand's OWN
  `section-*` spacing family (`compose_section.band_height_rung/band_height_css`,
  emitted as a `var(--space-…)` reference; wrapper stamps `data-band-height` +
  `data-band-rung` so the spacing auditor reads the deliberate re-registration).
  Documented in `brand-schema.md` §5.6 + `spec/archetype-library.md` §6b.
- **`requiresOffGrid`** (data-side flag in `heroes-saas.yaml`, no schema bump) —
  archetypes whose anatomy needs off-grid geometry devices; `shortlist(off_grid=False)`
  demotes them so refless/on-grid lanes never see them.
- **`archetypeRef`** (composition.v1 optional section key) — the recorded skeleton
  choice; drives skeleton-first normalization (`archetype_library.apply_archetype_skeleton`,
  brand recipes win by ordering), the `data-archetype` wrapper stamp, and the HARD
  `archetype-physics:<section>` onbrand rows.

Deferred (not load-bearing for these 8 heroes):

- `code-panel` block contract — no gallery hero renders a code surface (terminal-split
  excluded as un-instantiable; search-first took the developer slot).
- `typeAsGraphic` license — `hero-ghostword-atmospheric` was never selected (off-grid
  demotion + page-type affinity), so no hero needed the oversized-type-as-art license.
- `reducedMotion` contract entry — the gallery introduces no new motion devices; the
  existing interaction-audit reduced-motion posture already covers composed pages.
- `seamOffset` — the seam-straddle archetypes (`banded-seam-straddle`,
  `pricing-plan-peek`) were not selected, so no seam-registration length was needed.

## Gate scoping on novel skeletons (documented, not silently skipped)

- **onbrand (composition mode)** — sections stamped `data-archetype` audit in
  creative-mode scope: surface identity re-scopes from "the source layout's surface"
  to "one of the brand's OWN surface roles"; the source-heading-snippet cell re-scopes
  to "every authored display heading renders"; accent fidelity checks accent
  deployment rather than the source section's accent placement. NEW HARD
  `archetype-physics:<section>` rows map each of the archetype's `physicsBindings`
  to the row that enforces it (internal: alignment-resolution / rhythm / text-contrast /
  occlusion / interaction-contrast; `delegated[...]`: spacing_audit container+measure,
  slop AS-32/51/59/60/61/44/50/26, interaction_audit — all run by this lane, below).
- **alignment** — a genre-skeleton section takes structure from the archetype + the
  brand's OWN header-context grammar; the seeded pattern's alignment fact no longer
  rides through (`compose_section.resolve_alignment` skips the pattern layer for
  archetype sections; the stamp records `data-align-source="brand"`). Patterns keep
  donating treatments/rhythm.
- **spacing** — pads audit against the stamped `data-band-rung` (the knob's deliberate
  re-registration); a donor pattern's measured `bandPadding` is suppressed for
  archetype sections (band geometry belongs to the skeleton + the brand's ladder —
  `stamp_pattern_devices`); the hero media collage subtree is excluded from
  container-law candidacy (col-span registered media geometry, e.g. span 8 ≈ 896px,
  not expressible on the width ladder by construction — the acting content column
  audits instead, as stack-measure where the pattern declares one).
- **slop + interaction** — full battery, no scoping: all AS rules incl. 59/60/61 at
  1440+1180; interaction strict static+behavioral.
